# Plano de testes de usabilidade

O objetivo deste plano de testes é avaliar a facilidade de uso, compreensão da interface e eficiência das principais funcionalidades do SupplyAcoustic. Os testes foram planejados com base nas histórias de usuário definidas durante a etapa de especificação do projeto, buscando validar se os usuários conseguem executar as tarefas propostas sem auxílio externo.

Foram selecionados participantes com perfis semelhantes às personas definidas no projeto, incluindo estudantes de arquitetura e usuários com conhecimentos em tecnologia. Os testes serão realizados remotamente por compartilhamento de tela, utilizando o método de observação direta das ações executadas pelos participantes.

Durante a execução dos testes serão coletados os seguintes indicadores:

- Taxa de sucesso da tarefa (concluiu ou não concluiu);
- Tempo necessário para conclusão da tarefa;
- Quantidade de erros cometidos;
- Quantidade de cliques necessários;
- Necessidade de ajuda do avaliador;
- Nível de satisfação do usuário em escala de 1 a 5.

A escala de satisfação utilizada será:

1. Péssimo
2. Ruim
3. Regular
4. Bom
5. Ótimo

Em conformidade com a Lei Geral de Proteção de Dados (LGPD), nenhum dado pessoal dos participantes será registrado ou divulgado no relatório.

---

## Cenários de teste de usabilidade

| Nº do cenário | Descrição do cenário |
|---------------|----------------------|
| 1 | Você é um arquiteto e deseja criar um novo projeto acústico. Cadastre um ambiente informando suas dimensões e materiais utilizados. |
| 2 | Você deseja analisar um ambiente previamente cadastrado. Execute uma análise acústica e visualize os resultados de RT60 gerados pelo sistema. |
| 3 | Após realizar uma análise, visualize o relatório acústico gerado e identifique as recomendações de melhoria sugeridas pela plataforma. |
| 4 | Utilize o chat contextual da inteligência artificial para tirar dúvidas sobre os problemas acústicos identificados no ambiente analisado. |
| 5 | Consulte a quantidade de créditos disponíveis em sua conta e verifique se há créditos suficientes para realizar uma nova análise. |

## Métricas coletadas

| Métrica | Descrição |
|----------|-----------|
| Taxa de sucesso | Percentual de usuários que concluíram a tarefa |
| Tempo de execução | Tempo necessário para concluir a tarefa |
| Número de erros | Quantidade de erros cometidos durante a execução |
| Número de cliques | Quantidade de interações realizadas |
| Necessidade de ajuda | Indica se o participante precisou de auxílio |
| Satisfação | Avaliação subjetiva do usuário em escala de 1 a 5 |
